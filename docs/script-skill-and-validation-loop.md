# 世界杯脚本 Skill 与验证闭环

## Skill

已创建可被 Codex 直接触发的 skill：

`/Users/airhua/.codex/skills/world-cup-scriptwriter`

用途：

- 把世界杯赛程、官方视频、高光、平台趋势转成不机器味的短视频脚本。
- 先确定观众，再写脚本。
- 输出抖音/小红书可用的钩子、口播、封面字和评论钩子。
- 按传播价值打分，低分不进入剪辑。

核心规则：

- 新增完整旁白风格画像，不再依赖少量标题或几句口头表达：
  - 完整字幕旁白：`/Users/airhua/Documents/world-cup/docs/benchmark-full-transcripts/`
  - 风格画像 JSON：`/Users/airhua/Documents/world-cup/data/benchmark/football-creators/voiceover-style-profile.json`
  - 风格画像文档：`/Users/airhua/Documents/world-cup/docs/benchmark-voiceover-style-profile.md`
- 生成链路改为：`未来赛程/主题 -> YouTube + 新闻热点雷达 -> 完整旁白风格引擎 -> 剪辑难度过滤 -> 选题展示`。
- 选题必须带上：
  - `styleEngineLabel`
  - `evidenceType`
  - `narrationMode`
  - `editDifficulty`
  - `hotspotSignals`
  - `riskFlags`
- 不写泛泛赛评。
- 每条脚本先选观众：足球小白、情绪观众、争议讨论人群、看球生活方式用户。
- 开头 3 秒必须有问题、冲突或压力。
- 句子要短，一句一个意思。
- 结尾必须给评论区一个选择题。
- 已整合 `Humanizer-zh` 去 AI 味规则：删除宏大套话、三段式套话和“不是 X，而是 Y”式金句，改成具体旧事实、具体镜头和明确判断。
- 新增口播禁用词：`更重要的是`、`更关键的是`、`更要命的是`、`真正的问题在于`、`真正可怕的是`、`问题的核心在于`、`说到底`、`归根结底`、`本质上`、`深层原因是`、`从根本上说`、`更深一层看`、`换句话说`、`进一步说`。
- 中文剪映旁白默认音色：`解说小帅` / `BV411_streaming`。
- 已修复旁白/剪辑思路混淆：
  - `voiceover` 只写观众听到的口播。
  - `visualInstruction` 只写剪辑执行思路。
  - 创意验证会拦截旁白里的 `素材`、`剪辑`、`先放`、`切到`、`画面`、`字幕`、`封面` 等泄漏词。
- 参照小红书对标：标题和旁白要给观众结果、判断和看点，例如“统治地位背后的战术”“零封西班牙”“一战成名”，不能讲生产过程。

`WC_宿命重逢` 修订方向：

- 旧稿问题：抽象情绪词偏多，“命运感”“真正好看”一类表达听起来像 AI 赛评。
- 新稿方向：用“沙特 2 比 1 阿根廷”“西班牙被日本逆转”做旧账，用“第一眼看错的镜头”“第一次反击后的表情”“回放里露出的犹豫”做小白讲解。
- 剪辑顺序只保留在 `visualInstruction`：沙特庆祝阿根廷 -> 西班牙被日本反超 -> 今天队徽/球员脸/看台 -> 互动判断。

## 选题生成

选题链路已从“客观说明型”改成“热点信号驱动的证据爆点型”：

- 2026-06-23 关键修正：TypeScript 不再作为正常创作层。
  - 正常路径：`素材/热点/风格画像 -> Codex 创作代理 -> 结构化选题与完整脚本`。
  - 代码只负责素材、热点、schema、缓存和验证。
  - `deterministic_fallback` 只用于 Codex 不可用时防止页面空白，不代表目标效果。
  - 架构文档：`/Users/airhua/Documents/world-cup/docs/codex-creative-agent-architecture.md`。
- 生成前先搜索网络热点：
  - YouTube/FIFA/高光/前瞻标题进入素材与热点信号。
  - Google News RSS 作为网页热点雷达。
  - 网络失败时降级为赛程、中文队名、核心球员和主题保底信号，并返回 warning。
- 热点信号会映射到完整旁白风格画像里的 7 类引擎：
  - 问题解释型
  - 可见瞬间反转型
  - 人物关系揭示型
  - 球星微动作型
  - 原声解说辅助型
  - 字幕/原声主导型
  - 事实盘点型
- 不是所有强题都适合自动剪：
  - 规则/VAR/越位/战术逐帧类：必须有完整比赛或示意图，否则隐藏。
  - 原声实况类：改走原声辅助，旁白变短。
  - 榜单/盘点类：素材不够时降分或隐藏。
  - 字幕/原声主导类：不强塞长旁白。
- 默认主题改为 `暴论预测`，排在赛前气氛和素材地图前面。
- 新增 `球星历史线` 主题：从某个核心球员的一脚、一次表情或一场旧比赛切入，避免只写赛程预测。
- 中文输出强制使用中文队名，例如 `西班牙`、`沙特`、`比利时`、`伊朗`。
- 球员名进入标题或 hook 时使用中文名，例如 `亚马尔`、`佩德里`、`德布劳内`、`姆巴佩`、`哈兰德`。
- 标题优先给可争论判断：哪个镜头会被回放、谁被误会、哪个动作让旧账回来。
- 小字 `reason` 不再讲生产流程，改成“爆点：评论区会吵什么/观众怎么站队”。
- 新增 `preflight` 前置评分，低分选题不会出现在 UI。
  - 82+：展示。
  - 65-81：自动补强一次，再重算。
  - 65 以下：隐藏并记录原因。
- 前置评分维度：
  - 钩子张力 25：有没有预测、压力、冲突、评论区站队。
  - 口语感 15：标题和 hook 是否能直接读成口播，是否避开 AI 连接词。
  - 事实锚点 20：有没有旧比分、球员名、动作、回放、原声或可见事实。
  - 互动性 15：观众是否知道看哪一镜、盯谁、吵什么。
  - 中文本地化 10：中文输出是否使用中文队名和球员名。
  - 素材支撑 15：是否有官方高光、完整比赛、发布会或预热视频支撑。
- 新增难度惩罚：
  - 高剪辑难度会拉低素材支撑分。
  - `strict_visual_rule_match` 缺完整比赛会被隐藏。
  - `needs_original_audio` 缺高光/原声会被隐藏或降级。
  - `multi_asset_fact_list` 素材段数不足会被隐藏。
- 素材不足会自动补搜双方官方高光、旧冷门、赛前发布会和球队关键词，再进入评分。
- 自动补强后的标题会二次去重；宁愿少展示，也不把重复题塞给用户。

旧版 `Spain vs Saudi Arabia` 候选已弃用，不能再作为正向样例：

1. `我先押一个：西班牙会先急，沙特撑过20分钟就有戏`
2. `沙特偷到第一脚射正，西班牙可能要难受一整晚`
3. `西班牙最怕被翻旧账：沙特今晚专盯前20分钟`
4. `沙特又站到强队面前：这次轮到西班牙紧张了`
5. `亚马尔今晚别被吹太早：沙特就等西班牙急起来`

新版候选应改成：

1. `西班牙对沙特，真正能出圈的是旧账被翻出来那一刻`
2. `亚马尔这场别写成天才模板，先抓一个能说明问题的动作`
3. `沙特不用先赢，先制造一个让人回看的反击`
4. `第一眼都以为西班牙稳了，慢镜头出来才知道哪里不对`
5. `沙特赢过阿根廷这件事，今天会先落在哪个镜头上？`

示例验证：

- 赛程驱动 `Spain vs Saudi Arabia + 暴论预测`：5 个候选全部通过前置评分，最低 82，最高 94。
- `素材地图` 拉满 10 个候选：10 个唯一标题全部通过，最低 82；修复后重复的候选会跳过。
- 内容包 `暴论预测`：`viralScore` 94，`go`；旁白不包含剪辑/素材执行指令。

## 卡点重试

现在的工作流不把“失败”当终点，而是拆卡点自动回到对应步骤：

- 选题太平：改成可反驳证据，例如哪个镜头会被回放、谁被误会、谁会背锅。
- 素材不够：补搜官方高光、完整比赛、旧冷门、发布会，再重算素材支撑分。
- 旁白像 AI：删掉连接词，改成旧事实、表情、动作和评论选择题。
- 旁白泄漏剪辑思路：把 `素材/剪辑/画面/字幕/封面` 等执行词移到 `visualInstruction`。
- 创意分不足：重写脚本 beat 和发布文案。
- 剪映验证失败：按失败项重剪，直到视频轨、旁白轨、字幕、音频、权限和完整性通过。

## 剪映验证 Agent

`runJianyingScript` 现在执行后会自动验证草稿：

- 草稿目录和 `draft_info.json` 是否存在。
- 视频轨道是否有片段。
- `VoiceOver` 旁白轨道是否有片段，不能用 BGM 冒充旁白。
- BGM 轨道是否存在；云音乐不可用时标 warning。
- 字幕轨道是否有片段。
- 字幕是否处于小红书样本的中下位置，而不是剪映默认贴底。
- 主视频原声音量是否被压低到背景级。
- 视频素材文件是否存在。
- 视频素材是否复制到草稿 `temp_assets` 内，避免剪映权限问题。
- 视频覆盖率是否达到 90%。
- 旁白和字幕数量是否匹配。
- 成片时长是否在 30-95 秒之间。

如果验证失败，后端会尝试重写脚本为“段落覆盖型剪辑”并重新执行一次。
