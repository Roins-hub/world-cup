# 调研拆解

## 视频方案的创作思路

视频标题为《世界杯撸钱大法 热点怎么批量做？我用Codex把流程做成了AI程序》。字幕和简介里反复强调一个核心判断：

1. 世界杯短视频最难的不是写稿，而是脚本和画面能不能对上。
2. 因此工作流必须改成“先找素材，再写脚本”。
3. 一场比赛进入系统后，先查赛程，再找历史交锋、完整比赛、高光、赛前发布会和球队相关素材。
4. 根据已验证素材发散 5-10 个选题。
5. 选定题材后生成短视频脚本、素材地图、标题、发布文案。
6. 批量化价值来自连续赛程、多语言、多账号，而不是单条视频的文案能力。

视频里的原型功能包括：输出语言选择、主题输入、赛程选择、生成选题、创建内容、标题、热度评分、时长建议、脚本、素材地图、用户历史、管理员模型/API 配置、积分和生成记录。

## 技术路径

- 素材层：`yt-dlp` 搜索 YouTube，优先 FIFA 官方频道，分类为 full match / highlight / preview / press。
- 选题层：素材支撑度影响热度评分，避免产生没有画面的脚本。
- 内容层：脚本按 hook、context、conflict、proof、relevance、close 分段，每段绑定素材和视觉指令。
- 生产层：内容包保存为 JSON，同时导出 SRT，交给剪映脚本使用。
- 剪映层：用 `jianying-editor-skill` 的 `JyProject`、`add_media_safe`、`add_narrated_subtitles`、`add_text_simple` 生成草稿。

## 剪映 skill 能力边界

已阅读 `SKILL.md`、`docs/agent-playbook.md`、`docs/minimal-command-sop.md`、`rules/setup.md`、`rules/core.md`、`rules/cli.md`、`rules/media.md`、`rules/text.md`、`rules/audio-voice.md`、`rules/generative.md` 和核心源码。

可用能力：

- 本地/云端素材导入。
- 视频切片与多轨排列。
- TTS 旁白，优先剪映 SAMI，失败后可走 edge-tts。
- 字幕与旁白对齐。
- 花字、普通文字、动画。
- BGM 和音效轨道。
- 草稿检查与自动导出脚本。

重要限制：

- 业务剪辑脚本不能写进 skill 目录，必须放当前项目。
- 自动导出主要支持 Windows + 剪映 5.9 或更低版本。
- macOS 当前更适合生成草稿，导出需要手动。
- 长视频分析前建议压到 30 分钟内、360p；当前项目先做素材检索与高光下载，完整比赛默认只做验证来源。

## 本轮落地结果

已实现的 Web 产品闭环：

1. 在创作台选择语言、自由主题或近期赛程。
2. 后端通过 `yt-dlp` 搜索 YouTube 真实素材。
3. 按素材类型生成选题候选和热度评分。
4. 点击选题生成内容包：脚本、素材地图、发布文案、成本估算。
5. 点击“生成剪映计划”导出 JSON、SRT、Python 草稿脚本。
6. 点击“下载高光并生成脚本”下载高光类 MP4。
7. 点击“执行剪映草稿”运行脚本，生成剪映草稿。

实际验收：

- 内容包命令行生成成功。
- 高光素材下载成功：3 个 MP4。
- 剪映草稿生成成功：`WC_下载验证`。
- 草稿结构：4 条轨道、42 个片段，其中视频 3 段，旁白 19 段，标题 1 段，字幕 19 段。
