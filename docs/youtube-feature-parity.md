# YouTube 方案功能对照

对照视频：《世界杯撸钱大法 热点怎么批量做？我用Codex把流程做成了AI程序》。

## 核心工作流一致性

| 视频里的能力 | 当前实现 | 状态 | 说明 |
| --- | --- | --- | --- |
| 输出语言选择 | 前端支持中文、英文、西语、法语 | 已达成 | 和视频一致。 |
| 主题/赛程输入 | 改为“近期赛程下拉 + AI 主题方向下拉” | 已增强 | 视频里既可输入主题也可选赛程；本项目按用户“不懂球也要制造内容”的场景，把自由输入降级为命令行兼容，前台以选择式为主。 |
| 生成 5-10 个选题 | 支持 3-10 个选题 | 已达成 | 选题会围绕已找到素材发散。 |
| 先找真实素材 | `yt-dlp` 搜 YouTube，优先 FIFA 官方素材 | 已达成 | 素材按 full match/highlight/preview/press 分类。 |
| 素材地图 | 内容包包含素材地图和剪辑建议 | 已达成 | 每条素材有角色、剪辑建议、验证说明。 |
| 完整比赛与高光区分 | 完整比赛用于核验，高光用于下载剪辑 | 已达成 | 默认不下载完整比赛，避免文件过大。 |
| 内容包：标题、热度、时长、脚本 | 内容包完整输出这些字段 | 已达成 | 脚本分为 hook/context/conflict/proof/relevance/close。 |
| 发布文案 | 输出标题、简介、hashtags、封面字 | 已达成 | 可继续扩展为多平台版本。 |
| 历史记录 | 本地 `runs/` 保存每次内容包和脚本 | 部分达成 | 目前是文件历史，未做用户后台列表 UI。 |
| 管理员后台模型/API 配置 | 暂未做 | 未达成 | 当前后端是本地启发式生成，可后续接 OpenAI/Gemini 配置页。 |
| 用户积分/账号管理 | 暂未做 | 未达成 | 当前目标是自用生产闭环，不做 SaaS 支付和积分。 |
| 生成成本估算 | 内容包包含成本估算 | 已达成 | 先按视频里的 6 次交互/约 0.003 美元口径展示。 |

## 相比视频的增强

- 前台不要求懂球：从赛程下拉开始，AI 主题方向也由后端根据赛程给出。
- 剪映链路打通：从内容包继续生成 SRT、下载高光、生成并执行剪映草稿脚本。
- 权限修复：下载素材会复制到剪映草稿 `temp_assets` 下，避免 macOS 剪映读取 `Documents` 路径失败。
- 平台策略扩展：补充小红书/抖音/微博/B站的内容策略文档。

## 当前差距

1. 真实 LLM 生成器还未接入模型配置页；当前是可控的本地生成逻辑。
2. 历史记录还没有后台 UI，只保存在 `runs/`。
3. 没有做用户、积分、支付、风控。
4. 自动导出仍受剪映 skill 限制：Windows + 剪映 5.9 最稳；macOS 侧重点是草稿生成。

## 验收结论

“素材优先 -> 选题 -> 内容包 -> 素材下载 -> 剪映草稿 -> AI 配音/字幕”主链路已跑通。若目标是 1:1 自用生产工具，当前已覆盖视频演示的核心价值；若目标是 1:1 SaaS，需要继续补用户后台、模型配置、积分和生成历史管理。
